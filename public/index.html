🔌 Internal Endpoints

The service runs at 127.0.0.1:5757. All endpoints accept ?url=FACEBOOK_URL.

GET/healthHealth check
curl "http://127.0.0.1:5757/health"
GET/scrape?url=FB_URLfdown.net · Puppeteer
Provider: fdown.net — headless Chromium. Returns direct fbcdn.net links.
curl "http://127.0.0.1:5757/scrape?url=https://www.facebook.com/reel/1234567890"
GET/scrape-sf?url=FB_URLsavefrom.net · Puppeteer
Provider: savefrom.net — stealth Chromium session.
curl "http://127.0.0.1:5757/scrape-sf?url=https://www.facebook.com/reel/1234567890"
GET/scrape-snap?url=FB_URLsnapsave.app · vm-decode · ~250ms
Provider: snapsave.app — pure HTTP + Node.js vm sandbox. No Puppeteer needed.
curl "http://127.0.0.1:5757/scrape-snap?url=https://www.facebook.com/reel/1234567890"

VPS (Ubuntu/Debian)

    
apt-get install -y chromium-browser
git clone https://github.com/Casper-Tech-ke/fb-scraper.git
cd fb-scraper
npm install puppeteer-core node-fetch
npm install -g pm2
pm2 start server.js --name fb-scraper
pm2 save && pm2 startup


    
    
Render

    
Go to render.com → New Web Service → Connect GitHub → select Casper-Tech-ke/fb-scraper

    
# Build Command
npm install puppeteer-core node-fetch

# Start Command
node server.js

# Environment: Node
# Add environment variable:
# CHROMIUM_PATH = /usr/bin/google-chrome-stable

    
⚠️ Puppeteer on Render requires the Puppeteer buildpack. Add it in Settings → Buildpacks. For the snap endpoint only (no browser), it works out of the box on any plan.


    
    
Koyeb

    
Go to koyeb.com → Create App → GitHub → select Casper-Tech-ke/fb-scraper

    
# Run Command
node server.js

# Port: 5757
# Plan: Nano (free) or Micro for Puppeteer
# For full Chromium support, use Docker deploy with a custom Dockerfile


    
    
Replit

    
Go to replit.com → Create Repl → Import from GitHub → paste https://github.com/Casper-Tech-ke/fb-scraper

    
# In Replit Shell:
npm install puppeteer-core node-fetch

# In .replit file:
# run = "node server.js"

# Note: Chromium/Puppeteer is NOT available on Replit free tier.
# Only /scrape-snap (snapsave.app vm-decode) will work without a browser.
# To use /scrape and /scrape-sf, upgrade to Replit Core.
