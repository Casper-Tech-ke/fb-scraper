🔌 Internal Endpoints

The service runs at 127.0.0.1:5757 and is used internally by the Next.js API layer. All endpoints accept ?url=FACEBOOK_URL.

GET /health Health check
No upstream provider — returns service status.
curl "http://127.0.0.1:5757/health"
GET /scrape?url=FB_URL fdown.net · Puppeteer
Provider: fdown.net — headless Chromium. Returns direct fbcdn.net links. HD detected via /m366/ in URL.
curl "http://127.0.0.1:5757/scrape?url=https://www.facebook.com/reel/1234567890"
GET /scrape-sf?url=FB_URL savefrom.net · Puppeteer
Provider: savefrom.net — stealth Chromium session. Returns quality-labelled download links and thumbnail.
curl "http://127.0.0.1:5757/scrape-sf?url=https://www.facebook.com/reel/1234567890"
GET /scrape-snap?url=FB_URL snapsave.app · vm-decode · ~250ms
Provider: snapsave.app — pure HTTP POST + Node.js vm sandbox. No Puppeteer. Returns JWT-proxied rapidcdn.app links (~60min expiry).
curl "http://127.0.0.1:5757/scrape-snap?url=https://www.facebook.com/reel/1234567890"

🚀 Self-Hosting


    
Run your own instance in minutes.


    
1. Clone & Install

    
git clone https://github.com/Casper-Tech-ke/fb-scraper.git
cd fb-scraper
npm install puppeteer-core node-fetch


    
2. Install Chromium (Ubuntu/Debian)

    
apt-get install -y chromium-browser


    
3. Start the Service

    
# Direct
node server.js

# With PM2 (recommended for production)
npm install -g pm2
pm2 start server.js --name fb-scraper
pm2 save
pm2 startup


    
4. Test It

    
curl "http://127.0.0.1:5757/health"
curl "http://127.0.0.1:5757/scrape-snap?url=https://www.facebook.com/reel/1234567890"
